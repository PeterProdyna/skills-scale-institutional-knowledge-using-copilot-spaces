# OctoAcme Project Management Docs

This directory is the entry point for OctoAcme's lightweight but structured project management process. The process spans initiation, planning, execution, release, and retrospective improvement, with clear ownership and repeatable team rhythms.

## Process overview

OctoAcme projects move through a consistent lifecycle:
1. **Initiation**: clarify business need, success metrics, stakeholders, timeline, and initial risks.
2. **Planning**: build a prioritized backlog, define acceptance criteria and Definition of Done, estimate scope, and map milestones/dependencies.
3. **Execution & tracking**: deliver in small increments, track progress and blockers, and maintain regular team demos/reviews.
4. **Release & deployment**: ensure CI/security checks pass, prepare release notes and rollback plans, run smoke checks, and verify deployment.
5. **Retrospective & continuous improvement**: capture lessons learned and track follow-up actions with owners.

Core roles include **Project Managers**, **Product Managers/Product Leads**, **Developers**, **QA/Test Engineers**, **Engineering Managers**, **Designers/UX Researchers**, **Delivery Leads/Scrum Masters**, **Technical Leads/Architects**, **Stakeholders/Executive Sponsors**, **Support/Customer Success**, **Security/Compliance**, and **DevOps/Platform Engineers**. Communication follows a regular cadence (daily/twice-weekly standups, weekly syncs, sprint or milestone demos, and monthly stakeholder updates) with clear escalation paths for blockers, risks, and incidents.

Quality is built in throughout delivery: clear acceptance criteria, a documented Definition of Done, small pull requests linked to issues, CI checks (tests, linting, security scanning), manual QA when needed, release smoke tests, rollback readiness, and retrospective action tracking.
